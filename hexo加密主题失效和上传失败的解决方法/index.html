hexo加密主题失效和上传失败的解决方法

发表于 2021-02-07

and the repository exists.

以上是在hexo d上传到GitHub时的报错，最开始是网络问题，换了好的节点，改了端口2070解决了，今天又出现了这个问题，去搜了下解决方法，依次做了以下步骤：

1.删除.deploy_git文件夹，再hexo d && hexo g,重新部署，这一步骤完成后，hexo的加密主题开始正常显示，但是hexo d还是报错

2.添加git的环境变量，C:\Program Files\Git\bin，添加到path，仍然没有解决

3.http://chiva-zhao.github.io/2015/10/20/hexo-blog/ 在这篇文章中看到的解决方法，干掉相关的环境变量：”SVN_SSH” or “SSH”等含有TortoiseGit.exe的环境变量，我没有找到含有TortoiseGit.exe,但是在path里找到了名为%SYSTEMROOT%\System32\OpenSSH\的含有SSH的环境变量，删除后上传正常
